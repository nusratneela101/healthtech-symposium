## 1. প্রয়োজনীয়তা

শুরু করার আগে নিচের বিষয়গুলো নিশ্চিত করুন:

| প্রয়োজনীয়তা | বিবরণ |
|--------------|-------|
| **PHP 8.0+** | প্রয়োজনীয় এক্সটেনশন: `pdo_mysql`, `imap`, `openssl`, `mbstring` |
| **MySQL 5.7+** অথবা MariaDB 10.3+ | বেশিরভাগ cPanel হোস্টিংয়ে পাওয়া যায় |
| **cPanel হোস্টিং অ্যাকাউন্ট** | File Manager ও MySQL Databases অ্যাক্সেস সহ |
| **SSL (HTTPS) সহ ডোমেইন** | নিরাপদ কুকি ও OAuth-এর জন্য আবশ্যক |
| **Brevo অ্যাকাউন্ট** | [brevo.com](https://www.brevo.com/) — বিনামূল্যে ব্যবহার করা যায় |
| **Apollo.io অ্যাকাউন্ট** | [apollo.io](https://www.apollo.io/) — স্বয়ংক্রিয় লিড সংগ্রহের জন্য |
| **n8n অ্যাকাউন্ট** | [n8n.io](https://n8n.io/) Cloud অথবা নিজস্ব সার্ভারে ইন্সটল করা |

> **টিপস:** cPanel → **MultiPHP INI Editor** → আপনার PHP ভার্সন সিলেক্ট করুন → প্রয়োজনীয় এক্সটেনশনগুলো চেক করুন।

## 2. cPanel-এ ফাইল আপলোড

1. আপনার লোকাল মেশিনে পুরো প্রজেক্ট ফোল্ডারটি `.zip` করুন।
2. **cPanel**-এ লগইন করুন।
3. **File Manager** খুলুন এবং `public_html/` ফোল্ডারে যান (অথবা সাবডোমেইনের রুট ফোল্ডারে যান)।
4. **Upload** বাটনে ক্লিক করুন এবং `.zip` ফাইলটি আপলোড করুন।
5. আপলোড সম্পন্ন হলে `.zip` ফাইলটিতে রাইট-ক্লিক করুন এবং **Extract** নির্বাচন করুন।
6. নিশ্চিত করুন যে প্রজেক্টের সব ফাইল (যেমন `index.php`, `login.php`, `admin/`, `api/`, `n8n_workflows/` ইত্যাদি) সরাসরি `public_html/`-এর ভেতরে আছে — কোনো নেস্টেড সাবফোল্ডারে নয়।

> **গুরুত্বপূর্ণ:** যদি ফাইলগুলো `public_html/healthtech-symposium/`-এর ভেতরে চলে যায়, তাহলে File Manager-এর drag-and-drop বা **Move** অপশন ব্যবহার করে সেগুলো এক লেভেল উপরে নিয়ে আসুন।

## 3. cPanel-এ MySQL ডেটাবেস তৈরি

1. cPanel-এ **MySQL Databases**-এ যান।
2. **Create New Database**-এর অধীনে একটি নাম লিখুন (যেমন, `youruser_hts2026`) এবং **Create Database** ক্লিক করুন।
3. **MySQL Users → Add New User**-এর অধীনে একটি শক্তিশালী পাসওয়ার্ড দিয়ে নতুন ডেটাবেস ইউজার তৈরি করুন। ইউজারনেম ও পাসওয়ার্ড সংরক্ষণ করুন।
4. **Add User to Database**-এর অধীনে আপনার নতুন ইউজার ও ডেটাবেস সিলেক্ট করুন এবং **Add** ক্লিক করুন।
5. পরবর্তী পেজে **All Privileges** সিলেক্ট করুন এবং **Make Changes** ক্লিক করুন।
6. পরবর্তী ধাপে প্রয়োজন হবে বলে তিনটি মান সংরক্ষণ করুন:
   - ডেটাবেস নাম (যেমন, `youruser_hts2026`)
   - ডেটাবেস ইউজারনেম (যেমন, `youruser_dbuser`)
   - ডেটাবেস পাসওয়ার্ড

## 4. অটো-ইন্সটলার চালানো

প্রজেক্টে একটি ব্রাউজার-ভিত্তিক ইন্সটলার রয়েছে যা স্বয়ংক্রিয়ভাবে সমস্ত ডেটাবেস টেবিল তৈরি করে এবং ডিফল্ট ডেটা দিয়ে পূরণ করে।

1. ব্রাউজার খুলুন এবং নিচের URL-এ যান:
   ```
   https://yourdomain.com/install/auto_installer.php
   ```
2. ফর্মের ঘরগুলো পূরণ করুন:
   - **DB Host** — সাধারণত `localhost`
   - **DB Name** — ধাপ ৩-এ তৈরি করা ডেটাবেস নাম
   - **DB User** — ধাপ ৩-এ তৈরি করা ইউজারনেম
   - **DB Password** — ধাপ ৩-এর পাসওয়ার্ড
   - **App URL** — আপনার পূর্ণ ডোমেইন URL, যেমন `https://yourdomain.com` (শেষে `/` ছাড়া)
   - **Super Admin Email / Password** — লগইনের জন্য যে ক্রেডেনশিয়াল ব্যবহার করবেন
   - **Admin Email / Password** — একটি সেকেন্ডারি অ্যাডমিন অ্যাকাউন্ট (পরীক্ষার সময় একই তথ্য দিতে পারেন)
3. **Install Now** ক্লিক করুন।
4. ইন্সটলার যা করবে:
   - ডেটাবেস সংযোগ পরীক্ষা করবে
   - `install/setup.sql` থেকে সমস্ত প্রয়োজনীয় টেবিল তৈরি করবে
   - ইমেইল টেমপ্লেট ও অ্যাডমিন ইউজারসহ ডিফল্ট ডেটা যোগ করবে
5. সম্পন্ন হলে **Go to Dashboard** ক্লিক করুন।

> **ইন্সটলেশনের পরে:** অননুমোদিত পুনরায় ইন্সটলেশন রোধ করতে সাথে সাথে `install/` ফোল্ডারটি মুছে ফেলুন বা নাম পরিবর্তন করুন।
> File Manager-এ `install/` ফোল্ডারে রাইট-ক্লিক করুন এবং **Delete** নির্বাচন করুন, অথবা নাম পরিবর্তন করে `_install_done/` রাখুন।

## 5. `.env` ফাইল কনফিগার করা

`.env` ফাইলে সমস্ত সংবেদনশীল কনফিগারেশন মান (ডেটাবেস ক্রেডেনশিয়াল, API কী, SMTP সেটিংস) সংরক্ষিত থাকে। এটি কখনো ভার্সন কন্ট্রোলে কমিট করা হয় না।

### ধাপসমূহ

1. cPanel File Manager-এ প্রজেক্টের রুট ফোল্ডারে যান।
2. `.env.example` ফাইলটি খুঁজুন, রাইট-ক্লিক করুন এবং **Copy** নির্বাচন করুন।
3. কপির নাম দিন `.env`।
4. `.env`-এ রাইট-ক্লিক করুন এবং **Edit** নির্বাচন করুন।
5. নিচে বর্ণিত সমস্ত মান পূরণ করুন।

### সম্পূর্ণ `.env` টেমপ্লেট

```env
DB_HOST=localhost
DB_NAME=your_db_name
DB_USER=your_db_user
DB_PASS=your_db_password

APP_URL=https://yourdomain.com/fintech

SMTP_HOST=smtp-relay.brevo.com
SMTP_PORT=587
SMTP_SECURE=tls
SMTP_USER=your_brevo_login_email
SMTP_PASS=your_brevo_smtp_key
SMTP_FROM_EMAIL=info@yourdomain.com
SMTP_FROM_NAME=HealthTech Symposium

IMAP_HOST=imap.yourdomain.com
IMAP_PORT=993
IMAP_USER=info@yourdomain.com
IMAP_PASS=your_imap_password

BREVO_API_KEY=your_brevo_api_key

N8N_API_KEY=your_secure_random_key

SESSION_NAME=hts_session
```

### ভেরিয়েবলের বিবরণ

| ভেরিয়েবল | বিবরণ |
|----------|-------|
| `DB_HOST` | cPanel হোস্টিংয়ে সাধারণত `localhost` |
| `DB_NAME` | ধাপ ৩-এ তৈরি করা ডেটাবেস নাম |
| `DB_USER` | ধাপ ৩-এ তৈরি করা ডেটাবেস ইউজারনেম |
| `DB_PASS` | ধাপ ৩-এ দেওয়া ডেটাবেস পাসওয়ার্ড |
| `APP_URL` | অ্যাপটি যেখানে ডিপ্লয় করা হয়েছে তার পূর্ণ URL (শেষে `/` ছাড়া) |
| `SMTP_HOST` | Brevo SMTP হোস্ট: `smtp-relay.brevo.com` |
| `SMTP_PORT` | `587` (STARTTLS) |
| `SMTP_SECURE` | `tls` |
| `SMTP_USER` | আপনার Brevo অ্যাকাউন্ট লগইন ইমেইল |
| `SMTP_PASS` | আপনার Brevo SMTP কী (Brevo অ্যাকাউন্ট পাসওয়ার্ড নয়) |
| `SMTP_FROM_EMAIL` | আউটগোয়িং ইমেইলের "From" ঠিকানা |
| `SMTP_FROM_NAME` | আউটগোয়িং ইমেইলের "From" প্রদর্শন নাম |
| `IMAP_HOST` | ইনবক্স পোলিংয়ের জন্য IMAP সার্ভার হোস্টনেম (যেমন, `imap.yourdomain.com`) |
| `IMAP_PORT` | সাধারণত `993` (SSL-এর মাধ্যমে IMAP) |
| `IMAP_USER` | IMAP পোলিংয়ের জন্য ব্যবহৃত ইমেইল ঠিকানা |
| `IMAP_PASS` | IMAP অ্যাকাউন্টের পাসওয়ার্ড |
| `BREVO_API_KEY` | আপনার Brevo v3 API কী (ট্রানজ্যাকশনাল ইমেইল ট্র্যাকিংয়ের জন্য) |
| `N8N_API_KEY` | একটি শক্তিশালী, অনন্য সিক্রেট কী — সমস্ত n8n ওয়ার্কফ্লোতে ব্যবহৃত মানের সাথে **অবশ্যই মিলতে হবে** |
| `SESSION_NAME` | PHP সেশন কুকি নাম — পরিবর্তনের কারণ না থাকলে `hts_session` রাখুন |

> **নিরাপত্তা টিপস:** নিচের কমান্ড দিয়ে শক্তিশালী `N8N_API_KEY` তৈরি করুন:
> ```
> openssl rand -hex 32
> ```

## 6. Brevo সেটআপ (ইমেইল ডেলিভারি)

[Brevo](https://www.brevo.com/) (পূর্বে Sendinblue) ওপেন ও ক্লিক ট্র্যাকিং সহ নির্ভরযোগ্য ট্রানজ্যাকশনাল ইমেইল ডেলিভারি প্রদান করে।

### ধাপসমূহ

1. [brevo.com](https://www.brevo.com/)-এ নিবন্ধন করুন (বিনামূল্যে — কোনো ক্রেডিট কার্ড প্রয়োজন নেই)।
2. লগইনের পরে, **SMTP & API → API Keys**-এ যান এবং একটি নতুন API কী তৈরি করুন। কী-টি কপি করে সংরক্ষণ করুন।
3. **SMTP & API → SMTP**-এ যান এবং আপনার লগইন (ইমেইল ঠিকানা) ও SMTP কী নোট করুন।
4. আপনার `.env` ফাইলে এই মানগুলো আপডেট করুন:
   ```env
   SMTP_HOST=smtp-relay.brevo.com
   SMTP_PORT=587
   SMTP_SECURE=tls
   SMTP_USER=your_brevo_login_email
   SMTP_PASS=your_brevo_smtp_key
   BREVO_API_KEY=your_brevo_api_key
   ```
5. Brevo-তে **Senders & Domains → Domains**-এ যান এবং Brevo প্রদত্ত DNS রেকর্ডগুলো আপনার ডোমেইনে যোগ করে আপনার পাঠানো ডোমেইন যাচাই করুন। ডোমেইন যাচাই করলে স্প্যাম ফিল্টারিং উল্লেখযোগ্যভাবে কমে যায়।

> **মনে রাখুন:** `SMTP_PASS` হলো Brevo-র **SMTP কী**, আপনার Brevo অ্যাকাউন্টের লগইন পাসওয়ার্ড নয়। SMTP কী **SMTP & API → SMTP**-এ পাওয়া যায়।

## 7. Apollo.io সেটআপ (লিড কালেকশন)

[Apollo.io](https://www.apollo.io/) `lead_collector.json` n8n ওয়ার্কফ্লোর মাধ্যমে স্বয়ংক্রিয়ভাবে প্ল্যাটফর্মে লিড আমদানি করতে ব্যবহৃত হয়।

### ধাপসমূহ

1. [apollo.io](https://www.apollo.io/)-এ নিবন্ধন করুন।
2. **Settings → Integrations → API**-এ যান এবং একটি API কী তৈরি করুন। কী-টি কপি করে সংরক্ষণ করুন।
3. Apollo-তে আপনার প্রসপেক্টিং ফিল্টার সেট করুন:
   - **Location:** Canada
   - **Industry:** Health / Healthcare Technology

## 8. n8n সেটআপ (ওয়ার্কফ্লো অটোমেশন)

n8n লিড সংগ্রহ, ক্যাম্পেইন পাঠানো, ফলো-আপ ইমেইল এবং ইনবক্স পোলিং স্বয়ংক্রিয় করে। `n8n_workflows/` ফোল্ডারে পাঁচটি ওয়ার্কফ্লো JSON ফাইল সরবরাহ করা হয়েছে।

### ওয়ার্কফ্লো ফাইলের সংক্ষিপ্ত বিবরণ

| ফাইল | সময়সূচি | উদ্দেশ্য |
|------|---------|---------|
| `fintech_master_workflow.json` | প্রতি ৫ মিনিটে | নির্ধারিত ক্যাম্পেইন চেক করে → `api/send_one_email.php`-এর মাধ্যমে ইমেইল পাঠায় |
| `lead_collector.json` | প্রতিদিন সকাল ৮টায় | `api/save_lead.php`-এর মাধ্যমে Apollo.io থেকে লিড আমদানি করে |
| `followup_sender.json` | প্রতিদিন সকাল ১০টায় | যোগ্য লিডদের কাছে ফলো-আপ সিকোয়েন্স ২ ইমেইল পাঠায় |
| `response_tracker.json` | প্রতি ১০ মিনিটে | `api/poll_inbox.php`-এর মাধ্যমে IMAP ইনবক্স পোলিং করে রিপ্লাই সংগ্রহ করে |
| `thursday_campaign.json` | *(বাতিল)* | মাস্টার ওয়ার্কফ্লো + শিডিউলিং সিস্টেম দ্বারা প্রতিস্থাপিত — নিষ্ক্রিয় রাখুন |

### 8.1 n8n ইন্সটল করা

2. আপনার n8n ইন্সটেন্স URL এইরকম হবে: `https://yourname.app.n8n.cloud`

```bash
npm install -g n8n
```

**অপশন C — Docker দিয়ে**

```bash
docker run -it --rm \

  -p 5678:5678 \
  -v ~/.n8n:/home/node/.n8n \

```

### 8.2 ওয়ার্কফ্লো JSON ফাইল ইমপোর্ট করা

1. আপনার n8n ইন্সটেন্সে লগইন করুন।
2. বাম সাইডবারে **Workflows** ক্লিক করুন।
3. **+ New Workflow** (অথবা **+** বাটন) ক্লিক করুন।
4. উপরের ডান কোণে **⋮** (তিন-ডট) মেনু ক্লিক করুন এবং **Import from File** নির্বাচন করুন।
5. আপনার প্রজেক্টের `n8n_workflows/` ফোল্ডারে যান এবং `fintech_master_workflow.json` সিলেক্ট করুন।
6. বাকি `.json` ফাইলগুলোর জন্য ধাপ ২–৫ পুনরাবৃত্তি করুন:
   - `lead_collector.json`
   - `followup_sender.json`
   - `response_tracker.json`
   - `thursday_campaign.json` (ইমপোর্ট করুন কিন্তু **নিষ্ক্রিয় রাখুন**)

> **সমস্ত ওয়ার্কফ্লো `"active": false` দিয়ে ইমপোর্ট করা হয়** — ৮.৫ ধাপে ম্যানুয়ালি অ্যাক্টিভ না করা পর্যন্ত এগুলো চলবে না।

### 8.3 ক্রেডেনশিয়াল কনফিগার করা

ওয়ার্কফ্লো অ্যাক্টিভ করার আগে n8n-এ দুটি ক্রেডেনশিয়াল তৈরি করতে হবে।

#### Apollo.io — HTTP Header Auth

`lead_collector.json` Apollo API-তে কল অথেনটিকেট করতে ব্যবহার করে।

1. n8n-এ **Credentials** → **+ Add Credential** যান।
2. **HTTP Header Auth** খুঁজুন এবং সিলেক্ট করুন।
3. নিচের ঘরগুলো পূরণ করুন:

   | ঘর | মান |
   |----|-----|
   | **Name** | `Apollo API Key` |
   | **Header Name** | `X-Api-Key` |
   | **Header Value** | `<আপনার Apollo.io API কী>` |

4. **Save** ক্লিক করুন।
5. `lead_collector` ওয়ার্কফ্লো খুলুন, Apollo Search নোডে ক্লিক করুন এবং এই ক্রেডেনশিয়াল অ্যাসাইন করুন।

#### Brevo SMTP — অ্যাডমিন অ্যালার্ট ইমেইলের জন্য

নতুন রিপ্লাই আসলে `response_tracker.json` অ্যাডমিন নোটিফিকেশন ইমেইল পাঠাতে ব্যবহার করে।

1. n8n-এ **Credentials** → **+ Add Credential** যান।
2. **SMTP** খুঁজুন এবং সিলেক্ট করুন।
3. নিচের ঘরগুলো পূরণ করুন:

   | ঘর | মান |
   |----|-----|
   | **Name** | `Fintech SMTP` |
   | **Host** | `smtp-relay.brevo.com` |
   | **Port** | `587` |
   | **User** | আপনার Brevo লগইন ইমেইল |
   | **Password** | আপনার Brevo SMTP কী |
   | **SSL/TLS** | STARTTLS |

4. **Save** ক্লিক করুন।
5. `response_tracker` ওয়ার্কফ্লো খুলুন, **Send Alert Email** নোডে ক্লিক করুন এবং এই ক্রেডেনশিয়াল অ্যাসাইন করুন।

> **মনে রাখুন:** আউটবাউন্ড ক্যাম্পেইন ইমেইলগুলো সরাসরি PHP ব্যাকএন্ড `api/send_one_email.php` দিয়ে পাঠায় — n8n-এ সেগুলোর জন্য আলাদা SMTP ক্রেডেনশিয়াল প্রয়োজন নেই।

### 8.4 Placeholder রিপ্লেস করা

প্রতিটি ওয়ার্কফ্লো JSON-এ দুটি placeholder আছে যেগুলো আপনাকে আসল মান দিয়ে প্রতিস্থাপন করতে হবে:

| Placeholder | যা দিয়ে প্রতিস্থাপন করবেন |
|-------------|--------------------------|
| `YOUR_N8N_API_KEY` | আপনার `.env` ফাইলের `N8N_API_KEY`-এর মান |
| `YOURSITE.com` | আপনার আসল ডোমেইন, যেমন `yourdomain.com` |

**কীভাবে প্রতিস্থাপন করবেন:**

1. n8n এডিটরে প্রতিটি ওয়ার্কফ্লো খুলুন।
2. প্রতিটি **HTTP Request** নোডে ক্লিক করুন যেটি আপনার PHP API কল করে।
3. নোডের URL ঘরে `YOURSITE.com`-এর পরিবর্তে আপনার আসল ডোমেইন লিখুন।
4. **Headers** সেকশনে (বা query parameters-এ) `YOUR_N8N_API_KEY`-এর পরিবর্তে আপনার `.env`-এর আসল কী লিখুন।

> **টিপস:** ইমপোর্ট করার আগে কাঁচা JSON ফাইলগুলোতে আপনার টেক্সট এডিটরের **Find & Replace** ব্যবহার করুন — `YOUR_N8N_API_KEY` ও `YOURSITE.com`-এর সমস্ত occurrences প্রতিস্থাপন করুন, তারপর পরিবর্তিত ফাইলটি n8n-এ ইমপোর্ট করুন।

### 8.5 ওয়ার্কফ্লো অ্যাক্টিভ করা

1. n8n এডিটরে প্রতিটি ওয়ার্কফ্লো খুলুন।
2. নিশ্চিত করুন যে সমস্ত placeholder প্রতিস্থাপিত হয়েছে এবং ক্রেডেনশিয়াল অ্যাসাইন করা হয়েছে।
3. **Save** ক্লিক করুন।
4. এডিটরের উপরের ডান কোণে **Active** সুইচটি **On**-এ টগল করুন।
5. ওয়ার্কফ্লো স্ট্যাটাস ইন্ডিকেটর সবুজ হয়ে যাবে, যা নিশ্চিত করে যে এটি সক্রিয়।

**প্রস্তাবিত অ্যাক্টিভেশন ক্রম:**

1. `lead_collector.json` — লিড সংগ্রহ শুরু করুন
2. `fintech_master_workflow.json` — ক্যাম্পেইন পাঠানো সক্ষম করুন
3. `followup_sender.json` — ফলো-আপ অটোমেশন সক্ষম করুন
4. `response_tracker.json` — ইনবক্স পোলিং সক্ষম করুন
5. `thursday_campaign.json` — **অ্যাক্টিভ করবেন না** (বাতিল)

## 9. অ্যাডমিন প্যানেল পরিচিতি

সেটআপ সম্পন্ন হলে `https://yourdomain.com/login.php`-এ আপনার Super Admin ক্রেডেনশিয়াল দিয়ে লগইন করুন।

### ড্যাশবোর্ড

**URL:** `admin/dashboard.php`

ড্যাশবোর্ডে রিয়েল-টাইম পরিসংখ্যান দেখা যায়:
- **Total Leads** — ডেটাবেসে মোট লিডের সংখ্যা
- **Emails Sent** — সমস্ত ক্যাম্পেইনে পাঠানো মোট ইমেইল
- **Response Rate** — রিপ্লাই করেছেন এমন লিডের শতকরা হার
- **Conversion Funnel** — লিড স্টেজের ভিজ্যুয়াল বিভাজন

চার্টগুলো `/api/stats.php` এন্ডপয়েন্ট ব্যবহার করে স্বয়ংক্রিয়ভাবে আপডেট হয়।

- বাল্ক আপডেট করুন (যেমন, আগ্রহী হিসেবে চিহ্নিত করুন, যোগাযোগ না করার তালিকায় রাখুন)
- CSV-এ লিড এক্সপোর্ট করুন
- `admin/import_leads.php` দিয়ে ম্যানুয়ালি অথবা CSV বাল্ক আপলোডের মাধ্যমে লিড ইমপোর্ট করুন

**URL:** `admin/auto_campaign.php`

- একটি ইমেইল টেমপ্লেট

### শিডিউল ক্যাম্পেইন

**URL:** `admin/schedule_campaign.php`

**শিডিউলিং সিস্টেম কীভাবে কাজ করে:**

1. Super Admin অটো ক্যাম্পেইনে একটি ক্যাম্পেইন তৈরি করেন (`draft` হিসেবে সেভ হয়)।
2. Super Admin শিডিউল ক্যাম্পেইন খুলে ড্রাফট ক্যাম্পেইনটি সিলেক্ট করেন।
3. Super Admin datetime picker ব্যবহার করে একটি পাঠানোর তারিখ ও সময় বেছে নেন।
4. **📅 Schedule Campaign** ক্লিক করুন — ক্যাম্পেইনের স্ট্যাটাস `scheduled`-এ পরিবর্তিত হয়।
5. n8n-এর `fintech_master_workflow.json` প্রতি ৫ মিনিটে `api/get_scheduled_campaigns.php` পোল করে।
6. নির্ধারিত সময় এলে, n8n `api/send_one_email.php` লুপ করে সব ইমেইল পাঠায়।
7. সব ইমেইল পাঠানো হলে ক্যাম্পেইন স্বয়ংক্রিয়ভাবে `complete` চিহ্নিত হয়।

একটি নির্ধারিত ক্যাম্পেইন বাতিল করতে, শিডিউল তালিকায় পাশের **✕ Cancel** ক্লিক করুন।

> **গুরুত্বপূর্ণ:** `.env`-এ `N8N_API_KEY` আপডেট করলে, আপনার API এন্ডপয়েন্ট কল করে এমন প্রতিটি n8n ওয়ার্কফ্লোতেও একই মান আপডেট করতে হবে।

## 10. সমস্যা সমাধান

| সমস্যা | সমাধান |
|--------|--------|
| **ইন্সটলার DB-তে কানেক্ট করতে পারছে না** | `.env`-এ `DB_HOST`, `DB_NAME`, `DB_USER`, এবং `DB_PASS` দুবার চেক করুন |
| **500 Internal Server Error** | cPanel → **Errors**-এ PHP এরর লগ চেক করুন; MultiPHP-এ PHP 8.0+ নির্বাচিত আছে কিনা নিশ্চিত করুন |
| **ইন্সটলের পরে ব্ল্যাংক পেজ** | `.env`-এর `APP_URL`-এ শেষে `/` নেই এবং আসল URL-এর সাথে মিলছে কিনা নিশ্চিত করুন |
| **ইমেইল পাঠানো যাচ্ছে না** | `.env`-এর SMTP ক্রেডেনশিয়াল যাচাই করুন; আপনার হোস্ট পোর্ট 587 ব্লক করেনি কিনা নিশ্চিত করুন |
| **IMAP পোলিং কোনো রেজাল্ট ফেরত দিচ্ছে না** | cPanel → **MultiPHP INI Editor**-এ PHP `imap` এক্সটেনশন চালু করুন |
| **n8n `401 Unauthorized`** | n8n ওয়ার্কফ্লোতে `N8N_API_KEY` সার্ভারের `.env`-এর মানের সাথে মিলছে না |
| **n8n ক্যাম্পেইন পিক আপ করছে না** | নিশ্চিত করুন ক্যাম্পেইনের স্ট্যাটাস `scheduled` এবং `scheduled_at` অতীতে |
| **ক্যাম্পেইন `running`-এ আটকে আছে** | ম্যানুয়ালি রিসেট করুন: `UPDATE campaigns SET status='draft' WHERE status='running'` |
| **Apollo সার্চ ০ ফলাফল দিচ্ছে** | n8n-এ Apollo ক্রেডেনশিয়াল চেক করুন; API কী-তে People Search স্কোপ আছে কিনা যাচাই করুন |
| **CSS/JS লোড হচ্ছে না** | `APP_URL` সঠিক আছে কিনা নিশ্চিত করুন; `assets/css/style.css`-এ ফাইল অ্যাক্সেসযোগ্য কিনা দেখুন |
| **ওয়ার্কফ্লো ট্রিগার হচ্ছে কিন্তু নোডে এরর** | n8n-এ execution লগ খুলুন এবং ব্যর্থ নোডের আউটপুট পরীক্ষা করুন |
| **ওয়ার্কফ্লো শিডিউলে ট্রিগার হচ্ছে না** | নিশ্চিত করুন ওয়ার্কফ্লো **Active** আছে (টগল সবুজ) — শুধু সেভ করা যথেষ্ট নয় |
